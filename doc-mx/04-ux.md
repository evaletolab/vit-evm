# 04 — UX (iter 0.4, 2026-06-19)

## Routing & accès — guard wallet

- **`requireWalletGuard`** (`app/wallet/wallet.guard.ts`) — `CanActivateFn` qui lit `WalletStorageService.load()` et redirige (`UrlTree('/wallet')`) si aucun wallet en `localStorage['vit-wallet']`.
- Appliqué à toutes les routes **sauf** `/wallet` (carte = point d'entrée création/recovery) et `/claim` (atterrissage d'un lien de réception, doit pouvoir s'afficher pour expliquer au visiteur qu'il faut un wallet).
- Effet : avant qu'un wallet existe, toute navigation retombe sur la page Carte → user forcé de créer ou récupérer.

## Thème dynamique

**`ThemeService`** (`app/theme/theme.service.ts`) — service `providedIn: 'root'` qui :

- Définit 17 `ColorDef` couvrant toutes les vars CSS de `styles.scss` (`--bg`, `--surface-*`, `--glass*`, `--hairline*`, `--text*`, `--violet*`, `--cyan`, `--positive/negative/warning`), avec `alpha?` pour les vars rgba (verre, traits).
- Persiste l'override dans `localStorage['vit-settings']` (clé partagée avec devMode).
- `apply()` ré-écrit chaque var sur `document.documentElement` (rgba reconstruit via `hexToRgba`).
- **Gradients régénérés dynamiquement** : `--grad`, `--grad-neon`, `--grad-text` sont recomposés à partir des couleurs choisies (violet/violetDeep/cyan/text).

**Init** : `theme.init()` appelé dans `AppComponent` constructor, garantit l'application des overrides avant le 1er render.

**5 presets** (`COLOR_PRESETS`) : Lumen (défaut), Sunset, Forêt, Océan, Mono. Appliqués via `applyPreset()` qui patch toutes les clés visibles d'un coup.

**UI Profil → Réglages** (`page-account`) :

- Grille de tuiles de presets (mini-swatches 4 couleurs).
- 5 sliders visibles (`visible: true` sur le `ColorDef`) : **Fond droit** (`--surface-1`), **Fond gauche** (`--surface-2`), **Texte** (`--text`), **Bouton gauche** (`--violet`), **Bouton droit** (`--violet-deep`). Les 12 autres restent overridables côté code mais cachées du picker.
- Bouton « Réinitialiser » (clear localStorage entry + ré-applique defaults).

## Mode dev (gate Recevoir + logs)

- Toggle « Mode dev » dans Profil → Réglages, persisté dans `localStorage['vit-settings'].devMode`.
- Quand `devMode = false` (défaut) :
  - **Carte « Recevoir » (MockZCHF faucet)** masquée sur `/wallet` (`*ngIf="devMode"` sur l'article F2).
  - **Onglet « Recevoir »** retiré du tab-bar via getter `cardTabs` qui filtre l'array.
  - **4 blocs `<details class="debug">`** (`lastFaucet.debug`, `lastPayment.debug`, `addDeviceResult.op.debug`, `lastRecoveryOp.debug`) masqués → la page wallet reste « propre » pour un user end.
- Quand `devMode = true` : tout réapparaît. Permet de garder les outils de diagnostic en prod sans les exposer.

## IBAN affiché sur la carte (holo-card)

- La carte « Compte » du deck wallet (`page-wallet`) affiche une nouvelle ligne `IBAN` entre le numéro et le foot.
- Si `localStorage['vit-iban']` est set : affiche l'IBAN formaté en groupes de 4 (`formatIban()` regex `/(.{4})/g`).
- Sinon : lien `Configurer →` qui route vers `/iban`.
- Style : label mono 0.6rem opaque + valeur mono 0.78rem bold (cf. `holo-card__iban*` dans le scss).

## Wizard `/iban` — 4 étapes

`PageIbanComponent` est un wizard stateful (`step: 'checklist' | 'kyc' | 'enter' | 'done'`) + stepper visuel en tête de page.

1. **Préparer** — checklist statique (CI, caméra, justif domicile, téléphone) + adresse Safe copiable + CTA « Lancer la KYC ».
2. **KYC** — explication honnête : « Mt Pelerin n'a pas de page d'inscription séparée, le compte se crée via le widget d'achat (tape 10 CHF, l'email + KYC se déclenche au paiement) ». Tip block sur où trouver l'IBAN dans leur portail une fois validé.
3. **Saisir** — input grand format avec auto-format `CHxx xxxx xxxx …` (via `onIbanInput`) + **validation mod-97 ISO 13616** (`isValidIban`) avant `localStorage.setItem('vit-iban', clean)`.
4. **Fini** — carte de confirmation avec IBAN affiché + boutons Modifier/Effacer + carte Acheter/Vendre intégrée (toggle Buy/Sell + amount + CTA `window.open` widget).

Reprise d'état : si un IBAN est en localStorage au `ngOnInit`, on saute direct à `step = 'done'`.
